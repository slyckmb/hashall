# Stage Completion Report (QC-Oriented)

Write your report to `out/STAGE<N>-REPORT.md`.

## 1. Scope and Intent
- Stage goal(s) and what you aimed to complete.

## 2. Summary of Work Done
- What changed and why.

## 3. Files Changed
- List files touched with brief notes per file.

## 4. Tests Run
- Exact commands with pass/fail.
- Note any skipped tests and why.

## 5. Behavior Changes
- User-visible or data-model changes (or “None”).

## 6. Data / DB Actions
- What was deleted/archived/migrated, safeguards, approvals (or “None”).
- Any schema migrations added or applied.

## 7. UX Changes (if applicable)
- Before/after behavior and rationale (or “N/A”).

## 8. Decisions / Assumptions
- Important choices and rationale.

## 9. Requirements Alignment
- Docs reviewed and how work aligns with requirements.

## 10. Known Gaps / TODOs / Risks
- Anything incomplete or risky.
- Follow-up actions needed.

## 11. Artifacts / Outputs
- Any files generated (logs, reports, exports), with paths.

## 12. Rollback / Repro Notes
- How to revert or disable the change if needed.
- Any environment setup needed to reproduce results.

## 13. Open Questions
- Anything requiring owner decision before proceeding.
